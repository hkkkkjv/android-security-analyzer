# LLM-эксперименты: Итоги

## Сравнение: Статический анализатор vs ChatGPT 5.3

| Критерий | Статический анализатор | ChatGPT 5.3 |
|----------|----------------------|-------------|
| **Покрытие уязвимостей** | 6/8 (75%)  | 4/5 (80%)  |
| **Контекстные уязвимости** |  Детектирует (код + конфиг) | Пропускает |
| **Точное location** |  100% (файл:строка) | 0% |
| **Качество рекомендаций** |  Конкретные, но краткие |  Развёрнутые с примерами |
| **Воспроизводимость** |  100% (детерминировано) | ⚠~85% (вариативно) |
| **Ложные срабатывания** | 1 (заглушка в коде) | 1 (недоверие комментариям) |

---

## Вывод

> **Статический анализатор и LLM дополняют друг друга.**  
> Анализатор обеспечивает полное покрытие, точное локализацию и детерминизм — критично для CI/CD. LLM генерирует понятные объяснения и практические рекомендации — критично для разработчика.  
> **Оптимальная архитектура**: статический анализатор как основа детекции + LLM как постпроцессор для улучшения UX.
> **Улучшение**: статический анализатор требует улучшений с целью уменьшения ложных срабатываний
